# SOP: BackMarket Return (Refund/Replacement)

**Status:** Draft v1 (compiled from Ricky Q&A, 2026-04-06)
**Scope:** Customer returns a device bought from us on BackMarket for a refund or replacement.

---

## 1. Return Within 30 Days (Change of Mind)

- Device comes back in
- **Check warranty sticker** on bottom of device (confirms device hasn't been tampered with)
- Simple round of testing to confirm device is still in good condition
- Refund the customer
- Relist and sell again

## 2. Return Due to Fault

- Log what the fault is
- **Was it avoidable? Must be recorded every time.**
- Device goes through repair for the fault
- Then back through the listing process to resell

## 3. Replacement Request

- Sometimes the client wants a different device rather than a repair
- In those cases: swap bottom covers (changes the serial number)
- Relist as replacement unit

## 4. The Cost Stack (Why Returns Hurt)

Every return incurs:
- **Refund** to customer (full sale price)
- **10% BM commission** on original sale (already paid, gone)
- **Original outbound shipping cost** (gone)
- **Return shipping cost** (BM charges us for the return)
- **Re-repair cost** (time + parts if fault found)
- **Second 10% commission** when it sells again
- **Second outbound shipping cost**

Returns are extremely expensive. This is why:
- We try to convince clients to keep the device if possible
- Avoiding returns through better QC is critical
- Every return must log whether it was avoidable

## 5. Mandatory Logging

Same as warranty returns:
- What was the fault/reason?
- Was it avoidable?
- What caused it?
- What can we change to prevent it?
