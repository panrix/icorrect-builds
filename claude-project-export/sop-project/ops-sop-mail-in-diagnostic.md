# SOP: Mail-In Diagnostic

**Status:** Draft v1 (compiled from Ricky Q&A, 2026-04-06)
**Scope:** Clients who mail in a device for diagnostic investigation.

---

## Key Difference from Walk-In Diagnostic

**All information MUST be captured before the device arrives.** There is no face-to-face conversation to fill gaps. The pre-repair form is critical.

If information is missing when the device arrives → Ferrari contacts client → delay loop.

## Flow

1. **Booking and pre-repair form:** All diagnostic questions must be answered at booking (fault, cause, history, liquid damage details, data backup status, software version, passcode)
2. **Packaging sent to client** (same day, with return label)
3. **Confirm return address** - same or different from sender
4. **Device arrives:** Same intake checks as mail-in normal repair + ammeter reading
5. **Diagnostic process:** Same as [Walk-In Diagnostic](sop-walk-in-diagnostic.md) (Saf's full diagnostic flow)
6. **Diagnostic report and quote:** Ferrari writes and emails
7. **Client decision:** approve repair, decline, or request device back
8. **If approved:** repair → QC → ship back
9. **If declined:** ship device back unrepaired

## Turnaround
- Standard: 2-3 working days for diagnostic
- Faster option: within 24 hours (only if capacity allows)
- Must be confirmed at booking
