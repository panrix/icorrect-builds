# SOP: Mail-In Normal Repair

**Status:** Draft v1 (compiled from Ricky Q&A, 2026-04-06)
**Scope:** Clients who book a repair online and mail their device to us.

---

## 1. Order Comes In

- Client books via website (payment taken upfront at booking)
- Monday item created from booking

## 2. Packaging Sent to Client

- **Same day as order:** packaging is shipped to the client with their return label inside
- Client packs their device and sends it back

## 3. Device Arrives - Intake

When the device physically arrives:

1. **Confirm device matches** what client said they were sending
2. **Confirm payment taken** (check Monday)
3. **Test passcode** - confirm it's correct
4. **Power test** - does it turn on?
5. **Ammeter reading** - gives techs a baseline (same as walk-in: 5V = charging, 20V = front-end on MacBooks)
6. **Confirm the story matches** - does the device condition match what the client described?
7. **Confirm return address** - same as sender or different?
8. **Label the device** (same Brother label process by device type)

**Key difference from walk-in:** No face-to-face conversation. All information comes from the pre-repair form and booking details. Makes these intake checks even more critical since you can't ask the client on the spot.

If anything doesn't match or info is missing → tag Ferrari to contact client (delay loop).

## 4. Repair

Same flow as walk-in simple repair:
- Assigned to tech (Saf for most, Misha for iPad/MacBook screens)
- Tech checks notes, confirms repair, picks up part
- Repair → mid-repair test → seal → post-repair testing
- Update Monday notes → mark as Repaired → QC queue

## 5. QC (Ronny)

Same QC process as walk-in simple repair.

## 6. Post-QC: Packaging and Shipping

1. Device passes QC
2. Ronny packages the device for shipping
3. Nahid prints the return label and ships it out
4. Client receives notification that device is on its way

*(Need to verify Ronny/Nahid roles are still current)*

## 7. Warranty Returns

If a mail-in repair comes back with a fault:
- Log what the fault is
- **Was it avoidable?** Must be recorded
- Goes through warranty repair process
- Same "why did it happen" logging as all repair types

## Known Issues

1. No parts reservation/allocation at intake
2. Return address not always confirmed before shipping
3. All client communication for issues goes through Ferrari manually
