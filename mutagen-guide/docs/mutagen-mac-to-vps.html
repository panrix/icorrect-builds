Mutagen: Mirror a VPS to Your Local Mac

A step-by-step guide to installing Mutagen on macOS and setting up real-time bidirectional file sync with a remote Linux VPS over SSH.

Tested with Mutagen v0.18+ on macOS 13/14/15. Estimated time: 15–25 minutes.

1. What is Mutagen?

Mutagen is an open-source tool that gives you real-time, two-way file synchronisation between two endpoints — for example, your local Mac and a remote VPS over SSH. You edit files locally with your normal tools (VS Code, Cursor, Finder) and Mutagen mirrors every change to the server within milliseconds, and vice versa.

Key properties worth knowing up front:

  • rsync-like performance with low-latency filesystem watching on both sides.
  • No remote install needed. Mutagen ships a tiny "agent" binary to the VPS automatically the first time you connect (over scp).
  • Uses your existing SSH config. Keys, aliases in ~/.ssh/config, custom ports — it all just works.
  • Bidirectional or one-way sync modes, with conflict protection.

2. Prerequisites

Before you begin, make sure you have:

  • A Mac running macOS 11 (Big Sur) or newer.
  • Homebrew installed. If not, run the official installer at brew.sh.
  • SSH access to your VPS as a user with read/write permissions on the directory you want to sync.
  • The VPS hostname/IP, SSH port (usually 22), and your username.

Test SSH first. Open Terminal on your Mac and run ssh user@your-vps-ip. If that works, Mutagen will work.

3. Install Mutagen on macOS

1Install via Homebrew

Open Terminal on your Mac and run:

brew install mutagen-io/mutagen/mutagen
2Verify the install
mutagen version
mutagen daemon start

You should see a version number (e.g. 0.18.x). The daemon start command launches Mutagen's background process — you only need to run this once; macOS will keep it running and auto-start it on login.

Tip: To upgrade later, run brew upgrade mutagen followed by mutagen daemon stop && mutagen daemon start.

4. Set Up Passwordless SSH to the VPS

Mutagen reuses your existing SSH setup, so getting key-based auth working makes everything smoother (no password prompts during sync).

1Generate a key (skip if you already have one)
ssh-keygen -t ed25519 -C "your-email@example.com"

Press Enter to accept the default location (~/.ssh/id_ed25519). A passphrase is optional but recommended.

2Copy your public key to the VPS
ssh-copy-id user@your-vps-ip

If ssh-copy-id isn't available, append ~/.ssh/id_ed25519.pub manually to ~/.ssh/authorized_keys on the VPS.

3Optional — create a friendly SSH alias

Edit ~/.ssh/config on your Mac and add:

Host myvps
    HostName 203.0.113.42
    User ricky
    Port 22
    IdentityFile ~/.ssh/id_ed25519

Now ssh myvps connects without typing the full address. Mutagen will pick this alias up automatically.

5. Create Your First Sync Session

Mutagen sync URLs follow this format for SSH endpoints:

[user@]host[:port]:/absolute/path

The first endpoint is "alpha", the second is "beta". By default, sync is two-way and safe (conflicts pause the session rather than overwriting).

1Pick your local folder

Decide where on your Mac you want the mirror to live, e.g. ~/vps:

mkdir -p ~/vps
2Create the sync session

Using the SSH alias from step 4 (recommended):

mutagen sync create \
    --name=vps-mirror \
    --ignore-vcs \
    --ignore="node_modules" \
    --ignore=".DS_Store" \
    ~/vps \
    myvps:/home/ricky

Or without an alias:

mutagen sync create \
    --name=vps-mirror \
    --ignore-vcs \
    ~/vps \
    ricky@203.0.113.42:/home/ricky

The first time you run this, Mutagen copies a small agent binary to the VPS and starts watching both sides. You'll see the session move through scanning → reconciling → watching.

Heads up: If you point alpha at an empty local folder and beta at a populated remote folder, Mutagen will copy the remote files down to your Mac on first sync. If you want the opposite (or want to be sure), see the sync modes below.

Sync modes

Mode flagBehaviour
two-way-safe (default)Bidirectional. Conflicting edits pause the session for manual review.
two-way-resolvedBidirectional. On conflict, alpha wins.
one-way-safeAlpha → beta only. Won't delete unrelated files on beta.
one-way-replicaBeta becomes an exact mirror of alpha (will delete extra files on beta).

Add the mode with --sync-mode=<mode>, for example:

mutagen sync create --sync-mode=two-way-resolved --name=vps-mirror \
    ~/vps myvps:/home/ricky

6. Manage Sync Sessions

CommandWhat it does
mutagen sync listShow all sessions, their status, and any conflicts.
mutagen sync monitor vps-mirrorLive status display for one session.
mutagen sync flush vps-mirrorForce a full sync cycle now.
mutagen sync pause vps-mirrorPause a session (useful before doing bulk changes).
mutagen sync resume vps-mirrorResume a paused session.
mutagen sync terminate vps-mirrorPermanently end the session.

If a sync conflicts: run mutagen sync list --long. Mutagen will show which files conflict and on which side. Pick the version you want, delete the other, and run mutagen sync flush.

7. Optional: A mutagen.yml Project File

If you want to start/stop the whole sync setup with a single command (great for shared setups or multiple folders), drop a mutagen.yml into a project directory:

sync:
  defaults:
    ignore:
      vcs: true
      paths:
        - "node_modules"
        - ".DS_Store"
        - "*.log"

  vps-mirror:
    alpha: "~/vps"
    beta: "myvps:/home/ricky"
    mode: "two-way-safe"
    flushOnCreate: true

Then from that directory:

mutagen project start    # creates all sessions defined above
mutagen project list     # show project status
mutagen project terminate  # tear it all down

8. Troubleshooting

"Permission denied (publickey)"

Your SSH key isn't reaching the VPS. Test with ssh -v myvps. Re-run ssh-copy-id, or check that ~/.ssh/authorized_keys on the VPS contains your public key and has mode 600.

Session stuck in "Connecting"

Usually a firewall or wrong port. Confirm ssh user@host still works manually. If you use a non-default port, include it in the URL: user@host:2222:/path.

"unable to start session: agent installation error"

Mutagen needs to scp its agent binary to the VPS. Make sure the remote user can write to its home directory and that scp is installed (it is on virtually every modern Linux). Run scp testfile user@host:~ to confirm.

Sync is slow on first run

First sync hashes everything on both sides. A multi-GB folder can take a few minutes. Subsequent syncs are near-instant because Mutagen only ships the diffs.

Excluding more files

Add to the ignore.paths list in mutagen.yml, or pass extra --ignore=<pattern> flags. Patterns use .gitignore syntax.
